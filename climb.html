01

context

This project was completed during my summer 2016 internship at Bazaarvoice, over the course of about 4 weeks. Each intern at Bazaarvoice has a focal summer project, mine being the BV Analytics Project. This project was Andee's primary project.

The goal was to create a web-based interface for engineers to access a database to make easy changes. This database was previously only accessible via command line, which makes it time-consuming and inefficient to use.
Goal: Design an intuitive user interface for access to a rather complex database.
02

my role

My role in this project was to serve as the designer, including taking the reigns in user research, interface design, and experience design. I collaborated with another awesome intern, Andee, who served as the engineer. She built the functionalities of accessing, editing, and saving to the database.

Bazaarvoice is a company very big on working in an agile environment, so this project served as an awesome opportunity to practice agile, iterative development / design. My primary tasks included conducting light user research (light, given the relative simplicity of the target audience), designing intuitive and user-focused mockups, iterate based on standups with Andee, and implement the final design via HTML, CSS, and Javascript.
03

research

user personas
After speaking with the primary stakeholders - engineers using the DB, managers overseeing the DB - a couple personas and use cases were identified.

The primary user personas and use cases would be software engineers who are familiar with the EMOdb functions. Thus, the interface does not need much explanatory elements, but should ensure a simple and intuitive flow of information.

Affinity mapping coming soon
04

value prop

The importance of this project is the improvement of efficiency all across the engineering department. The time saved on training on how to use the database, actually using the database, etc. will be significant.

Not only will efficiency be greatly impacted, but intuitiveness and ability to make changes to the database will be improved. The web interface serves as not just simply an access to the database, but also shortcut deltas and conditionals that auto-wrap around the values.

06

results & takeaways

  • User-centric design: Since this project is for the sole purpose of simplifying a process for engineers, it was very important that the functionality and design did just that. This required iterative design - going through the cycle of wireframing, changes, updates - as well as better understanding the user needs through user research. Lesson learned, everything comes back to the user.

  • Research & analysis trumps design & production: Previously, I had always felt a much stronger emphasis on actually creating the design, actually implementing the design. However, it has become clear that the research and analysis stage will typically take longer, and is essentially the backbone to any further steps. Strong user research, accurate analysis of the data, and successful drawing of insights gives you the platform to create a strong design.
